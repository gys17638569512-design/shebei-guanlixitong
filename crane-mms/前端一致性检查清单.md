# 前端一致性检查清单

## 1. 文档信息
- **适用范围**：管理端 Web、客户端 Web、工人端微信/ Web
- **用途**：供总控智能体在联调、预验收、上线前做一致性检查
- **依据**：`PRD_重构版.md`、`账号体系与多端多智能体蓝图.md`、当前三端界面

## 2. 检查原则
- 三端统一品牌，但载体表达不同：Web 完整、微信核心。
- 页面风格统一，但操作场景允许差异化。
- 账号中心、品牌配置、主数据配置类页面优先保持一致。
- 所有新增页面先复用公共样式、公共卡片、公共表单、公共状态组件。

## 3. 品牌一致性
- [ ] 项目名称统一显示为 `数字化起重机维修维保系统`
- [ ] 三端页签、登录页、顶部导航、报告页标题保持统一口径
- [ ] Logo、系统色、辅助色、按钮主色保持一致
- [ ] 平台公司信息、客户公司信息、页脚版权文案可后台配置
- [ ] 报告页、登录页、空状态页不出现旧品牌名或临时占位名

## 4. 标题一致性
- [ ] 管理端页面标题格式统一，避免同一模块多种命名
- [ ] 客户端标题区分“客户服务门户”“个人中心”“设备详情”等语义
- [ ] 工人端标题区分“任务”“检查”“签字”“账号绑定”等语义
- [ ] 浏览器 `title`、页面主标题、面包屑标题保持一致

## 5. 按钮一致性
- [ ] 主操作按钮统一使用同一主色与同一圆角风格
- [ ] 次操作按钮、危险按钮、文字按钮样式统一
- [ ] 同义操作文案统一，如“保存”“提交”“确认”“退出登录”
- [ ] 禁用态、加载态、hover 态、点击态表现一致

## 6. 卡片一致性
- [ ] 列表卡片、详情卡片、账号卡片、统计卡片保持统一间距与阴影
- [ ] 卡片头部标题层级一致，辅助信息层级一致
- [ ] 空数据卡片与正常数据卡片保持同一视觉系统
- [ ] 卡片内操作按钮摆放规则统一

## 7. 表单一致性
- [ ] 表单标签、输入框、选择器、上传、日期等控件统一样式
- [ ] 校验提示文案统一，避免同一错误多种说法
- [ ] 新建/编辑/详情页保持同一字段顺序与命名
- [ ] 账号中心、品牌配置、客户公司、设备模板等高频表单优先复用同一表单骨架

## 8. 空状态一致性
- [ ] 无数据、无权限、加载中、网络失败四类状态样式统一
- [ ] 空状态文案简短明确，不使用技术化报错词
- [ ] 空状态插画/图标/按钮建议保持同一套视觉资源
- [ ] 列表、详情、弹窗、抽屉的空状态表现一致

## 9. 权限提示一致性
- [ ] 无权限提示统一口径，不同端口不出现不同语气
- [ ] 平台员工、客户主账号、子账号、工人账号的权限边界提示一致
- [ ] 禁用按钮、隐藏入口、跳转拦截的提示逻辑统一
- [ ] 微信核心版与 Web 完整版的能力差异要有明确提示

## 10. 账号中心检查
- [ ] 管理端员工账号中心风格统一，包含新建、改密、启停、绑定信息
- [ ] 客户端主账号/子账号管理风格统一，包含创建、授权、禁用、绑定信息
- [ ] 员工本人资料页与客户个人中心页的布局逻辑统一
- [ ] 微信绑定、手机号绑定、头像、昵称、登录密码的展示与编辑规则统一

## 11. 三端专项检查
- **管理端**
  - [ ] 左侧导航、顶部栏、内容区、统计卡片、表格、弹窗统一
  - [ ] 员工账号中心、品牌配置、主数据配置入口风格统一
- **客户端**
  - [ ] 登录、首页、设备列表、报告、签字、账号中心统一
  - [ ] 客户主账号与子账号页面风格统一
- **工人端**
  - [ ] 登录、任务列表、检查表、签字页、个人中心统一
  - [ ] 现场高频动作保持轻量、清晰、可操作

## 12. 验收输出要求
- [ ] 总控智能体在每次联调后标记本清单完成情况
- [ ] 若发现样式不统一，需回派对应端口子智能体整改
- [ ] 若发现品牌名、标题、权限提示不统一，需先修正后再进入功能验收
- [ ] 本清单与 PRD、蓝图文档同步更新，不允许长期脱节

## 13. 当前结论
- 该清单用于约束“看起来像同一个产品”的基础体验。
- 未来新增页面默认先过本清单，再进入功能验收。
