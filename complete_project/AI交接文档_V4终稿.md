## 一、项目背景与架构

### 业务背景
郭永盛（老郭）运营起重机维保业务，自研SaaS管理系统。

### 三类用户角色
| 角色 | 说明 | 使用端 |
|---|---|---|
| 平台方（郭氏维保） | 超级管理员，不受授权限制，有独立超级后台 | PC管理后台 |
| 业务发展商 | 其他维保公司，购买使用权 | PC管理后台 |
| 终端客户 | 工厂/企业，查看报告和设备状态 | 客户门户（Web） |
| 工程师 | 郭氏员工，现场作业 | 微信小程序 |

### 三种角色权限（管理后台内）
| 角色 | 首页视角 | 数据范围 |
|---|---|---|
| 管理员(ADMIN) | 全局视角，可切换三种视图 | 全部数据 |
| 业务经理(MANAGER) | 只看自己视角 | 只看manager_id=自己的客户和工单 |
| 工程师(TECH) | 只看自己视角 | 只看assigned_engineer=自己的工单 |

**重要：后端接口必须做数据隔离，不能只靠前端隐藏。**

### 4.1 首页工作台

**页面结构**
- 顶部：页面标题 + 日期 + 视角标签（管理员视图/业务经理视图/工程师视图）
- 管理员可切换三种视角预览；业务经理和工程师只看自己对应视角

**ADMIN视角内容**
- 统计卡片行（5张）：本周待处理 / 进行中 / 本月完工 / 逾期未处理（红色边框）/ 设备预警
- 逾期工单列表（左宽）：左侧红色竖条标记，显示客户+设备+类型+工程师+计划日期+状态标签+异常标签
  - 可点击工单行 → 跳转工单详情
  - "查看全部 →" 链接 → 跳转工单中心列表
- 工程师动态（右窄）：卡片/列表切换，显示今日工单数+本周完工+当前状态（待出发/作业中/今日完工/无单）
  - 点击工程师卡片 → 跳转该工程师工单列表（筛选）
- 设备临期预警（左下）：按设备/按客户切换Tab，显示设备名+客户+剩余天数+「安排维保」按钮
  - 「安排维保」按钮 → 打开新建派单抽屉（预填客户+设备，优先级自动设紧急，橙色提示条）
- 完工趋势折线图（右下）：近30天完工趋势

**右下角FAB快捷菜单**
- 点击蓝色+号展开：新建派单 / 安排维保 / 新增客户
- 再次点击或点击其他区域收起

**MANAGER视角** — 同ADMIN但数据隔离到自己负责的客户/工单
**TECH视角** — 只显示分配给自己的工单，简化版

### 4.2 工单中心

#### 工单列表页
**筛选工具栏**
- 搜索框（工单编号/客户名/设备名）
- 筛选：工单状态下拉 / 工程师下拉 / 工单类型下拉 / 日期区间选择器
- 右侧：+ 新建派单（主按钮）

**表格列**：工单编号（蓝色可点击）/ 客户名称（蓝色可点击）/ 设备名称（蓝色可点击）/ 工单类型标签 / 负责工程师 / 计划日期（逾期显示红色+逾期天数标签）/ 状态标签 / 操作

**工单状态标签**：待处理（橙）/ 进行中（蓝）/ 待签字（紫）/ 已完成（绿）/ 已作废（灰）

**逾期工单**：整行浅红色背景

**操作列（hover显示）**：查看详情图标 / 编辑图标
- 点「查看详情」→ 工单详情页
- 工单编号、客户名、设备名均可点击跳转对应详情页

#### 工单详情页（4种状态）

**待处理状态**
- 页头：工单编号+状态标签+优先级+操作按钮组（转派/改期/作废）
- 基础信息卡：客户名（蓝色可跳转）/ 设备名（蓝色可跳转）/ 工单类型 / 计划日期 / 工程师 / 创建时间
- 检查项预览卡：显示本次应检查的项目清单（来自检查模板）
- 时间线：工单创建记录

**进行中状态**
- 增加：工程师GPS打卡记录（到达时间+坐标）
- 实时状态更新

**待签字状态**
- 增加：检查结果列表（每项：名称/结果/异常备注/异常照片缩略图）
- 等待客户签字区域

**已完成状态**
- 完整检查结果（所有检查项+结果+异常详情）
- 现场照片（缩略图网格，点击大图预览）
- 客户签字图片
- 「查看报告」按钮 → 打开报告预览抽屉
- 「下载PDF」按钮 → 直接下载

**操作弹窗**
- 转派弹窗：选择新工程师（显示当前工单数）+ 转派原因
- 改期弹窗：日期选择器 + 改期原因 + 是否通知工程师
- 作废弹窗：作废原因（必填）+ 红色警告说明影响

#### 新建派单抽屉（三步流程）

**三种入口与预填逻辑**
| 入口 | 来源页面 | 预填内容 | 提示条颜色 |
|---|---|---|---|
| 普通新建 | 工单列表「+新建派单」 | 无预填 | 无提示条 |
| 预警预填 | 首页「安排维保」 | 客户+设备+优先级=紧急 | 橙色 |
| 客户预填 | 客户详情「新建工单」 | 仅客户 | 绿色 |
| 设备预填 | 设备详情「新建工单」 | 客户+设备 | 蓝色 |

**Step 1：基本信息**
- 所属客户（必填，下拉搜索）
- 选择设备（必填，联动客户过滤，下拉选项只显示设备名称）
  - 选客户后显示该客户设备数提示
  - 选设备+工单类型后展示检查项预览（来自检查模板）
- 工单类型（必填，来自字段设置→工单类型，含自定义项）
- 优先级（普通/紧急，双选项卡样式）
- 计划日期（必填）

**Step 2：安排人员**
- 工程师选择（四格卡片，显示今日工单数，空闲工程师绿色）
- 工单备注（选填，文本域）

**Step 3：确认派单**
- 信息摘要卡（工单信息+人员安排）
- 蓝色提示：确认后发送微信+短信给工程师

**提交成功** → 绿色成功页，显示工单编号，可关闭或查看详情
**校验**：每步点「下一步」时验证必填项，缺少时字段变红边框

## 五、设备编号体系（核心设计，必须理解）

### 三个字段
| 字段名 | DB字段 | 说明 | 必填 |
|---|---|---|---|
| 出厂设备编号 | factory_no | 铭牌原厂编号 | 否 |
| 客户场内编号 | site_no | 客户内部叫法 | 否 |
| 系统设备编号 | system_no | 全局唯一，同时作为二维码 | 是（自动生成）|

### 系统编号生成逻辑
```
前缀（字段设置配置，默认GS）+ 发展商ID（4位）+ 年份（4位）+ 后缀
后缀规则：
  - 有出厂编号 → 使用出厂编号（去除特殊字符转大写）
    示例：GS-0001-2024-QD50001
  - 无出厂编号 → 自动递增序号（位数在字段设置配置）
    示例：GS-0001-2024-000001
  - 手动覆盖 → 用户完全自定义（警告：修改后旧二维码失效）
```

### UI展示规则
- 新建设备时：系统编号区域有绿色（自动）/ 蓝色（有出厂编号）/ 橙色边框（手动）三种状态
- 设备详情Hero区：三编号横条，三格并排，各有颜色区分
- 设备列表：三编号独立三列，可通过「列显示」控制显隐
- 未填写的编号显示"— 未填写"灰斜体，不显示空白

### 14.1 原始PRD需求中有但文档未覆盖的功能

#### 维修工单（独立于维保工单，V3完全遗漏）
这是一个独立的工单流程，不是维保工单的子集：
- **触发场景**：维保工单中发现需要维修的问题，或客户直接发起报修
- **费用单据**：
  - 选择更换的配件（标准物料库下拉）→ 系统自动带出单价
  - 输入数量 → 自动计算金额
  - 填写人工费和其他费用 → 汇总总价
- **客户签字分支**：
  - 正常签字 → 确认维修事实和费用
  - 「仅确认维修事实（费用待审）」分支 → 工单状态标为「费用待确认」→ 推给业务经理处理
- **后端需要独立的维修工单模型**（不是现有WorkOrder）

- 不等报告生成完，即时通知
- 后端需要独立的安全隐患推送逻辑
